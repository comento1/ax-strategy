# 롯데웰푸드 AX 전략 구체화 웹 서비스 — 설계 문서

## 1. 서비스 목적

- **대상**: 팀장/매니저 직책자
- **목표**: 임원이 정의한 AI 활용 영역을 팀의 실제 워크플로우에 기반해 **실행 가능한 과제**로 전환하고, 조별 협의를 거쳐 **과제 정의서**까지 도출
- **성격**: 사전과제(개인) → 본 워크숍(조별)까지 **동일 서비스**에서 진행

---

## 2. 전체 서비스 플로우

```
[진입] 워크숍 코드 입력 → 조 번호/이름 선택 (본 워크숍 시)
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  사전과제 (개인)                                                  │
│  임원 영역 리뷰 → 1개 영역 선택 → 워크플로우 분석 → 과제 후보 도출  │
│  + 강사 질문 사항 수집                                            │
└─────────────────────────────────────────────────────────────────┘
         │ 제출 후 본 워크숍 입과
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  세션 1 — 임원 영역 → 실행 가능 과제로 전환                        │
│  워크플로우 상호 공유 → 현실성 검토(정량 평가) → 복수 구현 주제 확정  │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  세션 2 (선택) — 직책자 관점 추가 과제                             │
│  Track A: 세션1 확장 아이디에이션  |  Track B: 신규 과제 발굴       │
│  우선순위 설정, 워크플로우 분석                                    │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  세션 3 — 과제 정리 및 인계                                       │
│  전체 과제 리스트업 → 환경조성(데이터/보안/인력) → 과제정의서 작성    │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
     과제정의서 (다운로드/인계)
```

## 3. 정량 평가 설계 (현실성 검토)

구두 논의가 아닌 **정량 평가**를 위해 아래 3문항을 1~10점으로 부여하고, 필요 시 ICE 방식으로 종합 점수를 쓴다.

| 평가 항목 | 질문 | 설명 (1=낮음, 10=높음) |
|-----------|------|------------------------|
| **전략 부합도 (Impact)** | 임원진이 규명한 조직의 문제를 해결하는가? | 1: 무관 ~ 10: 핵심 과제와 직접 부합 |
| **구현 가능성 (Ease)** | 사내 보안/인프라 환경 내에서 구현이 가능한가? | 1: 불가능에 가까움 ~ 10: 현재 환경으로 가능 |
| **데이터 확보성 (Confidence)** | 필요한 데이터가 확보되어 있거나, 확보할 수 있는가? | 1: 없음/불명 ~ 10: 확보 완료 또는 계획 명확 |

- **ICE 점수**: (전략 부합도 + 구현 가능성 + 데이터 확보성) ÷ 3 → **1~10** (평균)  
  또는 (I × C × E) 사용 시 1~1000 범위로 차이를 크게 둘 수 있음.  
  **권장**: 평균값으로 1~10 유지해 “우선 구현할 과제” 순위에 활용.

세션1에서 과제 후보별로 위 3항목을 조원이 각자/합의로 입력하고, **과제별 ICE 점수**로 정렬해 우선순위를 정한다.

## 4. UI/UX 설계 원칙

1. **직관적·명확한 구조**
   - 상단에 **단계 탭**: 사전과제 | 세션1 | 세션2 | 세션3
   - 현재 단계만 강조, 이전 단계는 보기 전용으로 이동 가능
   - 한 화면에 한 가지 “할 일”이 보이도록 블록 구분

2. **사전과제**
   - 첫 화면에 **수행 가이드 4단계** 고정 노출 (무엇을·어떻게 할지 명시)
   - 임원 AI 영역: **목록 + 1개 선택**만 하면 다음 단계로
   - 워크플로우: **단계 추가/순서 변경** 가능, 단계별 “AI 적용 가능” 여부 태그
   - 과제 후보: **추가/삭제/순서**만으로 목록 관리
   - 강사 질문: **텍스트 입력 + 제출** 한 곳에 모음

3. **세션1**
   - “우리 조” 결과만 보이도록 **조 단위 뷰**
   - **워크플로우 공유**: 조원별 선택 영역·워크플로우 요약 카드
   - **현실성 검토**: 과제 후보별 3항목(전략 부합도/구현 가능성/데이터 확보성) 입력 → ICE 점수 자동 계산, 표로 정렬
   - **구현 주제 확정**: ICE 순으로 상위 N개 선택 또는 체크박스로 확정

4. **세션2 (선택)**
   - **Track 선택** (A: 임원 범위 확장 / B: 신규 과제) 후, 해당 트랙만 표시
   - 신규/확장 과제도 **동일한 워크플로우·과제 후보·ICE** 구조로 입력 가능
   - 세션1 확정 주제 + 세션2 과제를 **하나의 “과제 풀”**로 관리

5. **세션3**
   - **전체 과제 리스트** (임원 기반 + 신규) 한 목록
   - 과제별 **환경조성**: Key Man, 데이터 확보 전략, 보안 이슈 등 짧게 입력
   - **과제정의서**: 양식에 맞춰 자동 채움 → PDF/워드 다운로드 또는 인쇄

6. **저장·동기화**
   - 모든 단계 **자동 저장** (로컬 또는 조별 저장소)
   - 하단에 “마지막 저장 시각” 표시
   - 본 워크숍 시 **워크숍 코드 + 조**로 진입 시 해당 조 데이터만 표시

## 5. 데이터 구조 (요약)

- **워크숍**: 코드, 이름, 임원 AI 영역 목록(시드)
- **조**: 워크숍 내 조 번호/이름, 소속 참가자(선택)
- **참가자**: 이름, 사전과제(선택 영역, 워크플로우, 과제 후보, 질문 목록)
- **조 공유**: 세션1 확정 구현 주제, 과제별 ICE 점수, 세션2 추가 과제, 세션3 과제 정의서 필드
- **과제정의서**: 과제명, 배경/목표, 범위, 워크플로우 요약, 제약·요건, Key Man, 데이터·보안 메모

## 6. 고객사 피드백 반영

| 피드백 | 반영 |
|--------|------|
| 무엇을·어떻게 해야 하는지 명확하고 직관적으로 | 상단 가이드, 단계별 한 블록씩, 버튼 라벨 명확화 |
| 현실성 검토를 구두가 아닌 매트릭스/평가도구로 | 3문항 + ICE 정량 평가 테이블 |
| 복수 주제 도출 시 샘플/가이드 | USE CASE 샘플 링크 또는 예시 과제 카드 제공 |
| 세션2는 조직별로 선택 운영 | Track A/B 선택 가능, 세션2 전체를 “선택”으로 표시 |
| 과제정의서 양식 구조만 잘 잡기 | 세션3에서 항목 고정, 출력 양식 통일 |

## 7. 기술 구현 방향

- **프론트**: 단일 페이지(SPA) 또는 단계별 HTML, 반응형
- **상태/저장**: 1차 localStorage (개인·데모), 추후 워크숍/조별 백엔드 API로 대체 가능
- **과제정의서 출력**: HTML → 인쇄/PDF 저장 또는 간단한 서버 PDF 생성
- **진입점**: URL 쿼리 `?workshop=코드&group=조번호` 로 워크숍·조 식별. 백엔드 연동 시 해당 조 데이터 로드/저장에 사용.

## 8. 구현 상태 (현재 버전)

- 사전과제: 영역 선택, 워크플로우 단계(추가/삭제/태그), 과제 후보, 강사 질문 수집
- 세션1: 워크플로우 요약, ICE 정량 평가 테이블, 구현 주제 확정 체크
- 세션2: Track A/B 선택, 확장·신규 과제 추가/삭제
- 세션3: 전체 과제 리스트, 과제정의서 폼, 인쇄/PDF 저장
- 저장: localStorage 자동 저장 (워크숍/조 코드는 URL 파라미터로 표시만, 동기화는 백엔드 확장 시)
